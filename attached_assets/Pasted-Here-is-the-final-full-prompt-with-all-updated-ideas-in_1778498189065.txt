```text
Build a minimal, offline-first mobile app called ResetFlow.

The app is a personal daily routine + study tracking app. It must be fast, smooth, practical, and not overloaded. It should not become Notion, Todoist, Quizlet, or a heavy productivity app.

The app must work fully offline:
- no account
- no cloud
- no internet dependency
- local storage only
- export/import data anytime for moving to another device

Main goal:
Help me not forget daily routines, temporary one-day tasks, and track study time separately using Pomodoro.

Important rule:
Daily routine tracking and study tracking must stay separate.

Bottom navigation:
- Routine
- Study
- Stats
- Settings

Optional later:
- Items / Checklists

The app must feel minimal, modern, and smooth. Avoid lag, excessive animations, heavy charts, too many screens, and unnecessary features.

==================================================
CORE MODULE 1: ROUTINE
==================================================

The Routine page is the main page.

It should show:

Monday, May 11
Today

Progress badge:
0/0 done or 4/8 done

Sections:
1. Daily Routine
2. Today’s Tasks

Daily Routine:
These are default daily tasks.
They repeat every day.
Their done status resets every day.
The task itself remains permanently unless deleted.

Examples:
- Drink water
- Shower
- Clean desk
- Prepare backpack
- Check email
- Sleep before 12:00
- Take vitamins
- Do laundry
- Review tomorrow’s schedule

Each daily task should support:
- name
- optional deadline/time limit
- done / not done
- category, optional
- created date
- edit
- delete

At the start of a new day:
- all daily tasks become not done again

Today’s Tasks:
These are temporary / variable tasks.
They are added for today only.
They disappear after the day ends unless moved to tomorrow.

Examples:
- Print visa document
- Ask teacher about assignment
- Buy shampoo
- Send email
- Charge power bank
- Submit homework

Each temporary task should support:
- name
- optional deadline/time limit
- done / not done
- created date
- edit
- delete

At the start of a new day:
- completed temporary tasks are removed
- unfinished temporary tasks should trigger an optional prompt:
  “Move unfinished tasks to today?”
  Options:
  - Move to today
  - Delete
  - Decide one by one

For MVP, it is acceptable to simply remove old temporary tasks, but the better implementation is the carry-to-tomorrow prompt.

Routine page empty states:

“No daily tasks yet”
“Add tasks that repeat every day”
Button: Add routine task

If no temporary tasks:
“No extra tasks today”
“These disappear at end of day”

For Daily Routine + button:
- task name
- optional deadline
- save

For Today’s Tasks + button:
- task name
- optional deadline
- save

Task display example:

- one tap checkbox/tick marks task as Done

- action should feel instant and satisfying
- avoid slow animations

Tasks with deadline should show:
- before 22:00
- due soon
- missed

Sorting:
Routine tasks should be sorted simply:
1. overdue/missed
2. due soon
3. normal unfinished
4. done

Top progress:
Show total completed tasks:
4/8 done

Optional small progress bar:
Today

==================================================
RESET LOGIC
==================================================

Daily routine reset:
When app opens:
1. Check current date.
2. Compare with lastOpenedDate.
3. If date changed:
   - set all daily tasks isDone = false
   - remove completed temporary tasks from previous days
   - ask whether to move unfinished temporary tasks to today, or delete them
   - update lastOpenedDate

This does not need a background service.

Study data should never reset.
Study sessions remain permanently unless user deletes them.

==================================================
PERFORMANCE REQUIREMENTS
==================================================

The app must not be laggy or overloaded.

Performance principles:
- local database should be fast
- avoid unnecessary re-rendering
- avoid loading all history into UI at once
- calculate stats efficiently
- cache summaries when needed
- keep animations light
- keep charts simple
- avoid huge libraries if possible
- avoid background processes unless necessary
- app should open quickly
- marking a task done should be instant
- starting timer should be instant
- switching tabs should be smooth

Recommended storage:
Use SQLite or a reliable local database.
For small MVP, local key-value storage is acceptable, but SQLite is better for long-term study sessions.

Efficient stats:
For MVP:
- calculate totals from StudySession records

For larger data:
- add cached daily/monthly summaries

Example summary table:
DailyStudySummary:
- date
- subjectId
- totalMinutes

MonthlyStudySummary:
- month
- subjectId
- totalMinutes

This prevents lag when calculating 1 year or all-time stats.

Important:
Even if the user has thousands of study sessions, Stats should remain smooth.

Charts:
- simple bar chart for last 7 days
- simple trend chart for 30 days
- avoid complex animations
- lazy load charts only on Stats page

UI:
- render only visible lists
- archive old subjects instead of deleting study history
- avoid excessive shadows/blur effects
- avoid heavy gradient animations
```
